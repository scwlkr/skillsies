Questions

FAQ rows test density, borders, and readable long text.

Keep these rows close to production-style content so spacing and contrast problems are obvious.

What should this visualizer prove?

It should prove that the chosen design tokens work across a realistic marketing/product page in both light and dark modes.

Where do the real design choices go?

Update the CSS token layer first, then adjust component details only when the token layer cannot express the decision cleanly.

Should this replace implementation?

No. This file is a visual agreement artifact. The app still needs its real components, data, routes, and accessibility checks.

Why keep generic copy?

Generic copy prevents the template from inventing product claims before the grilling session resolves the real language.
